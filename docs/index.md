# Phase 5. Custom Regression Project

For Phase 5, I applied the regression workflow to a new problem using the CDC Diabetes Health Indicators dataset.

Instead of predicting a student's final score from study-related features, I created a new regression problem: predicting **BMI** using health, lifestyle, demographic, and socioeconomic features.

This allowed me to apply the skills from the example project to a substantially different dataset and problem domain.

## Basis and Data

The original example project used a small synthetic dataset called `hours_scores_case.csv`. That dataset contained information about students, including study hours, practice quizzes, attendance, sleep, prior scores, and final scores.

For my custom project, I used the `diabetes.csv` dataset located in:

```text
data/raw/diabetes.csv
```

The dataset contains:

* **70,692 rows**
* **22 columns**
* **21 modeling columns after selecting BMI as the target**

The dataset includes health-related, lifestyle, demographic, and socioeconomic information.

I chose this dataset because it provided a substantially different problem from the original student-score example while still containing a continuous numeric variable that could be used for regression.

One limitation is that the dataset is observational. The model can identify relationships between the available variables and BMI, but the results should not be interpreted as proof that any individual factor causes a change in BMI.

## Modeling Approach

This is a **supervised machine learning** problem because the dataset contains a known target variable, `BMI`, that the model learns to predict from labeled examples.

This is a **regression** problem because BMI is a continuous numeric value rather than a category or class label.

The target variable is:

```text
BMI
```

The model used was:

```text
LinearRegression
```

Linear regression was appropriate for this project because the goal was to predict a continuous numeric value and to create a clear, interpretable baseline model.

The model used a 70/30 train-test split with:

```text
TEST_SIZE = 0.30
RANDOM_STATE = 42
```

The fixed random state makes the train-test split reproducible.

## Target

In the original example project, the target variable was:

```text
score
```

The model predicted a student's final score based on study and student-related features.

For my custom project, I changed the target variable to:

```text
BMI
```

BMI was selected because it is a continuous numeric variable and therefore fits a regression problem.

Changing the target changed the interpretation of the model. Instead of asking, "What final score might this student receive?", the custom model asks, "What BMI value might be predicted based on the available health, lifestyle, demographic, and socioeconomic features?"

The evaluation metrics also remain appropriate for regression. MAE, RMSE, and R-squared can be used to evaluate how closely the predicted BMI values match the actual BMI values.

## Features

The original example used the following student-related features:

* `hours_studied`
* `practice_quizzes`
* `attendance_pct`
* `sleep_hours`
* `prior_score`

For the custom project, I removed the original student features and selected 20 features from the diabetes dataset:

* `HighBP`
* `HighChol`
* `CholCheck`
* `Smoker`
* `Stroke`
* `HeartDiseaseorAttack`
* `PhysActivity`
* `Fruits`
* `Veggies`
* `HvyAlcoholConsump`
* `AnyHealthcare`
* `NoDocbcCost`
* `GenHlth`
* `MentHlth`
* `PhysHlth`
* `DiffWalk`
* `Sex`
* `Age`
* `Education`
* `Income`

These features were selected because they represent health conditions, health behaviors, physical and mental health, demographics, and socioeconomic factors that could potentially provide information related to BMI.

The target column, `Diabetes_binary`, was not used as the target for this project because I wanted to create a regression problem rather than a classification problem. I instead selected `BMI` as the continuous target variable.

## Evaluation and Results

I evaluated the model using three regression metrics:

### Mean Absolute Error (MAE)

```text
4.80
```

The MAE means that the model's predictions were off by approximately **4.8 BMI units on average**.

### Root Mean Squared Error (RMSE)

```text
6.59
```

The RMSE was higher than the MAE, which indicates that some prediction errors were larger than the average error.

### R-squared

```text
0.15
```

The R-squared value of 0.15 means that the model explained approximately **15% of the variation in BMI** using the selected features.

The model was also used to predict BMI for a new example case. The prediction was:

```text
Predicted BMI: 31.3
```

The result was interesting because the model successfully applied the regression workflow to a much larger and completely different dataset. However, the relatively low R-squared value also showed that successfully running a model does not necessarily mean that the model is a strong predictor.

BMI is influenced by many factors, and the available dataset does not include every factor that could affect the outcome. A future improvement could include trying other regression models, comparing multiple models, tuning model parameters, or performing additional feature analysis.

## Summary

For my custom project, I adapted the original regression workflow to a new dataset and a new prediction problem.

The original project predicted student scores using study-related features. My custom project used the CDC Diabetes Health Indicators dataset to predict BMI using 20 health, lifestyle, demographic, and socioeconomic features.

I implemented the following workflow:

1. Loaded the diabetes dataset.
2. Inspected the dataset structure and column names.
3. Checked for missing values and duplicate rows.
4. Selected BMI as the continuous target variable.
5. Selected 20 predictor features.
6. Created a clean modeling view.
7. Split the data into training and testing sets.
8. Trained a `LinearRegression` model.
9. Evaluated the model using MAE, RMSE, and R-squared.
10. Predicted BMI for a new example case.
11. Created visualizations showing predictions and residuals.

The final model produced:

* **MAE:** 4.80
* **RMSE:** 6.59
* **R-squared:** 0.15
* **Example predicted BMI:** 31.3

The most important thing I learned was how to transfer an existing machine learning workflow to a new problem. I also learned that choosing the target variable is an important part of defining the type of machine learning problem. Selecting BMI created a regression problem because BMI is a continuous numeric value.

This project exercised the skills covered in the regression workflow, including data loading, data inspection, data cleaning, feature selection, supervised learning, model evaluation, prediction, visualization, and interpretation.

These skills could be applied to many real-world problems involving continuous numeric predictions, such as predicting prices, energy use, sales, measurements, or other numeric outcomes.
